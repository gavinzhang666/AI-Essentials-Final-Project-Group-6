# 🏦 FinAnalyzer Pro 2026: Agentic Financial Audit Suite

**Project Authors:** Team 6 / Yuhui & Collaborators  
**Academic Context:** Graduate Research Project, 2026  
**Target Data:** SEC 10-K Filings (Amazon, Alphabet, Microsoft, etc.)  

---

## 🌟 Project Overview
FinAnalyzer Pro is a progressive Retrieval-Augmented Generation (RAG) suite designed to audit complex, unstandardized financial reports. This project demonstrates the architectural evolution from localized, privacy-focused models to advanced agentic cloud reasoning capable of bypassing common GAAP reporting inconsistencies.

---

## 🛠️ Version Architecture

### 1. **V1: Local Ollama (Sovereign Edition)**
- **File Name:** `v1_ollama_sovereign.py`
- **Technical Stack:** Ollama (`llama3.1`), `nomic-embed-text`, FAISS.
- **Design Philosophy:** Prioritizes absolute data sovereignty. By keeping the entire RAG pipeline on-premises, it ensures maximum privacy for sensitive financial data.
- **Key Feature:** Zero-data-leakage local inference.

### 2. **V2: HF Embeddings + Gemini 2.5 Flash (Hybrid Bridge)**
- **File Name:** `v2_hybrid_bridge.py`
- **Technical Stack:** HuggingFace (`BGE-Small-EN-v1.5`), Gemini 2.5 Flash, FAISS.
- **Design Philosophy:** Balances local embedding efficiency with world-class cloud reasoning.
- **Key Feature:** High-speed local indexing combined with professional-grade financial synthesis.

### 3. **V3: Gemini-Embedding-001 + Gemini 2.5 Flash (Intelligence Edition)**
- **File Name:** `v3_gemini_agent_pro.py`
- **Technical Stack:** `gemini-embedding-001`, Gemini 2.5 Flash, Agentic Query Expansion.
- **Design Philosophy:** Solves the "terminology gap" in financial reporting using a multi-query agent layer.
- **Key Feature:** **Agentic Query Expansion** — automatically translates user prompts into multiple accounting synonyms (e.g., mapping R&D to `Technology and infrastructure`) and performs manual sums for missing aggregates.

---

## 🚀 Technical Highlights

| Feature | Description |
| :--- | :--- |
| **Agentic Expansion** | Dynamically branches a single user query into 4+ targeted financial search terms to ensure data coverage. |
| **Local Persistence** | FAISS indices are saved locally, reducing API costs and enabling instant session recovery for multi-file audits. |
| **Table Integrity** | High-capacity chunking (2500 tokens) with 600-token overlap to prevent financial tables from being severed. |
| **Zero-Shot Calculation** | Intelligent reasoning logic to calculate "Total Operating Expenses" by summing constituents if the line item is missing. |

## ⚙️ Setup & Installation Instructions

### Part 1: General Environment Setup
Before running any version, ensure you have Python 3.10+ installed and set up the base environment:

1. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Configure Environment Variables:**  
   Create a `.env` file in the root directory and add your Google API key (required for V2 and V3):
   ```env
   GOOGLE_API_KEY="your_google_gemini_api_key_here"
   ```

### Part 2: Running Specific Versions

#### ▶️ How to run V1: Local Ollama
This version runs entirely locally and requires the Ollama background service.

1. Download and install Ollama.
2. Pull the required models via terminal:
   ```bash
   ollama pull llama3.1
   ollama pull nomic-embed-text
   ```
3. Launch the app:
   ```bash
   streamlit run v1_ollama_sovereign.py
   ```

#### ▶️ How to run V2: HF Embeddings + Gemini 2.5 Flash
This version uses local HuggingFace embeddings but calls Gemini for reasoning.

1. Ensure your `.env` file has a valid `GOOGLE_API_KEY`.
2. Launch the app (HuggingFace models will download automatically on first run):
   ```bash
   streamlit run v2_hybrid_bridge.py
   ```

#### ▶️ How to run V3: Gemini-Embedding-001 + Gemini 2.5 Flash
This is the flagship version utilizing the full Google Cloud stack for maximum intelligence.

1. Ensure your `.env` file has a valid `GOOGLE_API_KEY`.
2. Launch the app:
   ```bash
   streamlit run v3_gemini_agent_pro.py
   ```

**Recommended Test:** Compare Amazon, Alphabet, and Microsoft’s R&D as a percentage of OpEx to see agentic terminology resolution in action.
